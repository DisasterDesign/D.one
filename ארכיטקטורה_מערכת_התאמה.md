# ארכיטקטורה: מערכת התאמת הצעות מחיר מול חשבוניות

## סקירה כללית

מערכת ווב שרצה על האתר הקיים d-one1.com (Next.js + Vercel).
רואה החשבון מעלה שני קבצי Excel — אחד מ-Vevi Dental (שנהב), אחד מחשבשבת.
המערכת מבצעת Cross-Reference אוטומטי לפי **מספר עבודה** ומוציאה דוח מאומת.

## פירוט טכני

### 1. אימות (Authentication)

אימות פשוט מבוסס סיסמה — בלי רישום, בלי DB:

```
- סיסמה אחת שמורה ב-MATCHER_PASSWORD (env variable ב-Vercel)
- כניסה → API route בודק סיסמה → מחזיר JWT cookie (httpOnly)
- Middleware בודק cookie בכל גישה ל-/matcher/*
- Token תקף ל-7 ימים
```

**ספריות:** `jose` (JWT ללא תלויות)

### 2. פרסור קבצים

**שנהב (Vevi Dental Export):**
```typescript
interface ShenhavRecord {
  workNumber: string;      // מספר עבודה (המפתח!)
  clientName: string;      // שם לקוח/מרפאה
  description: string;     // תיאור העבודה
  amount: number;          // סכום
  date: string;            // תאריך
  status: string;          // סטטוס (in lab וכו')
}
```

**חשבשבת (Excel Export):**
```typescript
interface HashavshevetRecord {
  invoiceNumber: string;   // מספר חשבונית
  clientName: string;      // שם לקוח
  workNumber: string;      // מספר עבודה (מתוך שדה הפרטים!)
  amount: number;          // סכום
  date: string;            // תאריך
  type: string;            // סוג מסמך
}
```

**ספריות:** `xlsx` (SheetJS) לפרסור Excel

**חשוב:** הפרסור צריך להיות גמיש — מיפוי עמודות שניתן לשנות בקלות (config).

### 3. Matching Engine — לוגיקת ההתאמה

```
שלב 1: NORMALIZE
  - הסר רווחים, מקפים, אפסים מובילים ממספרי עבודה
  - "0036864" → "36864"
  - " 36864 " → "36864"

שלב 2: MATCH (Forward - הצעות → חשבוניות)
  לכל הצעה בשנהב:
    חפש חשבונית בחשבשבת עם אותו מספר עבודה
    אם נמצא → MATCHED ✅
    אם לא נמצא → MISSING_INVOICE ❌

שלב 3: REVERSE MATCH (חשבוניות → הצעות)
  לכל חשבונית בחשבשבת:
    חפש הצעה בשנהב עם אותו מספר עבודה
    אם לא נמצא → ORPHAN_INVOICE ⚠️

שלב 4: AMOUNT CHECK
  לכל זוג מותאם:
    אם סכום_הצעה ≠ סכום_חשבונית → AMOUNT_MISMATCH 💰
    (כולל אחוז הפרש)

שלב 5: DUPLICATE CHECK
  בדיקה: האם יש מספרי עבודה כפולים?
  אם כן → DUPLICATE_WARNING ⚡
```

**זו הבדיקה הכפולה (Double Verification):**
- Forward: כל הצעה → יש חשבונית?
- Reverse: כל חשבונית → יש הצעה?
- זה מבטיח שאין פספוסים בשני הכיוונים

### 4. דוח תוצאות

**תצוגה במסך:**
```
┌────────────────────────────────────────┐
│ סיכום                                   │
│  ✅ 142 הצעות עם חשבונית                │
│  ❌ 8 הצעות ללא חשבונית                 │
│  ⚠️ 3 חשבוניות ללא הצעה                │
│  💰 5 הפרשי סכומים                      │
│  ⚡ 0 כפילויות                          │
│                                         │
│  [הורד דוח Excel]  [הורד דוח PDF]       │
└────────────────────────────────────────┘
```

**דוח Excel מיוצא:**
- גיליון 1: סיכום כללי
- גיליון 2: הצעות ללא חשבונית (❌) — **הכי חשוב**
- גיליון 3: חשבוניות ללא הצעה (⚠️)
- גיליון 4: הפרשי סכומים (💰)
- גיליון 5: כל ההתאמות (✅)
- גיליון 6: Log מלא — כל שורה מכל קובץ עם הסטטוס שלה

## אבטחה

- סיסמה ב-Environment Variable (Vercel)
- JWT httpOnly cookie
- קבצים לא נשמרים בשרת — עיבוד בזיכרון בלבד
- Rate limiting על API routes
- הגבלת גודל קובץ (10MB max)

---

## נקודות קריטיות לאמינות

1. **Fuzzy matching** על מספר עבודה — טיפול ברווחים, אפסים מובילים, מקפים
2. **בדיקה דו-כיוונית** — Forward + Reverse
3. **Log מלא** — כל שורה מכל קובץ מופיעה בדוח עם סטטוס
4. **שורות לא מזוהות** — שורות שלא ניתן לפרסר מהן מספר עבודה → מסומנות בנפרד
5. **סיכום נומרי** — סה"כ שורות בקובץ מקור = סה"כ שורות בדוח (וידוא שלמות)
6. **חותמת זמן** על כל דוח

## Deployment

הכל על ה-Vercel הקיים:
1. Push לאותו repo של d-one1.com
2. הוספת env variable: MATCHER_PASSWORD + JWT_SECRET
3. Deploy — אוטומטי דרך Vercel
4. הנתיב /matcher נגיש מיד
